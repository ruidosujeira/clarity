Less noise, more signal.

Clarity executes the real command, captures stdout/stderr in ~/.clarity/logs, and returns a calm block with status, warnings, and next steps. Keep using npm, git, docker, or anything else—just without the wall of text.

Why Clarity?

Smart summaries

Plugins for npm, git, docker, bun, go, rust, and more recognize patterns and spotlight what matters.

Logs first

Every run gets a timestamped file. Open it with clarity logs --open, send to your pager, or jump into your editor.

Profiles on demand

Toggle between the restrained calm persona and the assertive devops mode with --profile.

No lock-in

The original command and exit code stay intact. Need the full stream? Run with --full or --raw.

Up and running in 60s

npm install -g clarityterm
clarity npm install
clarity logs --limit 3 --open

Contributing? Clone the repo, run npm install, and use npm link to exercise the local binary.

Ready for your stack

Ship it as an npm package, call node ./bin/clarity, or wrap any command you trust. Tune environment variables like CLARITY_PROFILE, CLARITY_LOGS_DIR, and CLARITY_PAGER to match your workflow.
